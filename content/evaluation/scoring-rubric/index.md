## 어떻게 점수를 매기나

모든 솔루션은 두 가지 축에서 평가한다. 각 축은 10점 만점이다.

- **A축: 일반 기능** — 제품 자체의 완성도
- **B축: 표준 정합성** — Zero Trust 표준과의 일치도

종합 점수는 두 축의 평균이다.

## 평가 항목 ID 체계

모든 항목에 고유 ID가 있다. 외부에서 정확히 인용하고 검색할 수 있게 하기 위함이다.

```
ZTA-{Pillar}-{Function}-{NN}
```

| 구성 | 의미 |
|---|---|
| Pillar 코드 | `ID` (Identity), `DEV` (Devices), `NET` (Networks), `APP` (Applications), `DATA` (Data), `VIS` (Visibility), `AUTO` (Automation), `GOV` (Governance) |
| Function | 영역 안의 세부 기능 코드 (예: AUTH, STORE, RISK) |
| NN | 두 자리 순번 |

예시: `ZTA-ID-AUTH-03`은 Identity 영역의 Authentication 기능 중 3번 항목을 가리킨다.

## A축: 일반 기능 (10점)

제품을 도구로서 평가하는 축이다.

| 항목 | 보는 것 |
|---|---|
| A-1. 통합 관리 | 단일 콘솔, API, 멀티 테넌트 |
| A-2. 가용성 | SLA, 다중 리전, 재해 복구 |
| A-3. 성능 | 처리 용량, 지연, 확장성 |
| A-4. 운영성 | 학습 곡선, 문서, 한국어 지원 |
| A-5. 비용 투명성 | 가격 공개도, 모듈화, 한국 파트너 |

## B축: 표준 정합성 (10점)

NIST 7원칙과 CISA ZTMM 기준이다.

| 항목 | 보는 것 |
|---|---|
| B-1. 신뢰 없이 항상 검증 | 모든 접근을 검증하는가 |
| B-2. 최소 권한 | 권한을 최소로 부여하고 동적으로 조정하는가 |
| B-3. 침해 가정 | 침해를 가정하고 설계됐는가 |
| B-4. 지속 검증 | 세션 중에도 다시 검증하는가 |
| B-5. 컨텍스트 정책 | 기기, 위치, 위험을 정책에 반영하는가 |
| B-6. 전 구간 암호화 | 끝에서 끝까지 암호화하는가 |
| B-7. 가시성과 분석 | 로그와 이상 탐지가 되는가 |
| B-8. ZTMM 단계 | Traditional부터 Optimal 중 어느 단계인가 |
| B-9. 자동화 정합 | 정책 자동화와 IaC를 지원하는가 |
| B-10. 거버넌스 정합 | 감사 추적과 증적이 충분한가 |

## 점수 산정 방법

1. 항목마다 0점, 1점, 2점 중 하나 (커버 안 함, 부분 커버, 완전 커버)
2. 축별로 합산해서 10점 만점으로 환산
3. 종합 점수는 두 축의 평균

## 단계 표기

| 종합 점수 | 단계 |
|---|---|
| 9.0 이상 | Optimal |
| 7.0~8.9 | Advanced |
| 5.0~6.9 | Initial |
| 5.0 미만 | Traditional |

## 면책 사항

이 점수는 공개 자료를 근거로 한다. 실제 운영 환경의 성능과 운영성은 다를 수 있다.

벤더 간 1대1 비교에 단독으로 사용하지 않는다. 절대 평가이지 상대 평가가 아니다.

마지막 검토일 이후 사실관계가 달라졌을 수 있다.
