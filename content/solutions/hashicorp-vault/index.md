## 어떤 제품인가

HashiCorp Vault는 API 키, 비밀번호, 인증서 같은 **시크릿을 통제하는 도구**입니다. 시크릿에 대한 단일 인터페이스를 제공하면서 접근을 엄격히 통제하고 상세 감사 로그를 남깁니다([출처](https://github.com/hashicorp/vault)).

핵심은 **신원 기반 통제와 동적 시크릿**입니다. 데이터를 저장 전에 암호화하고, AWS나 SQL 같은 시스템에는 필요할 때 자격 증명을 즉석에서 생성한 뒤 리스 만료 시 자동 회수합니다([출처](https://developer.hashicorp.com/vault)). 시크릿, 사용자, 머신, 서비스, AI 에이전트의 수명 주기를 자동화해 수동 회전과 폐기에서 오는 사람 실수를 줄입니다([출처](https://www.hashicorp.com/en/products/vault)).

2025년 2월 IBM이 **64억 달러**에 HashiCorp 인수를 완료했습니다([TechCrunch](https://techcrunch.com/2025/02/27/ibm-closes-6-4b-hashicorp-acquisition/)).

| 모듈 | 역할 |
|---|---|
| 시크릿 관리 | 키와 인증서를 암호화 저장, 단일 인터페이스 제공 |
| 동적 시크릿 | 필요 시 자격 증명 생성, 리스 만료 시 자동 회수 |
| 암호화 서비스 | 저장 없이 암호화와 복호화 제공 |
| PKI | 인증서 수명 주기 관리, KMIP 연동 |
| Enterprise 네임스페이스 | 팀별 격리 환경과 중앙 정책 관리 |

## 강점과 한계

**강점**
- 수명이 짧은 동적 시크릿으로 최소 권한과 침해 영향 축소를 동시에 구현
- 암호화 서비스로 개발자가 암호화를 직접 설계하지 않아도 됨
- 상세 감사 로그로 시크릿 접근 가시성 확보
- 오픈소스 버전 제공으로 도입 진입 장벽이 낮음

**한계**
- 운영 복잡도와 학습 곡선이 커서 전담 인력이 필요
- 기기와 위험 컨텍스트 기반 정책은 상대적으로 제한
- IBM 인수 후 제품 로드맵 변화는 시장 관찰 필요
- 한국 CSAP 등 국내 인증 근거는 공개 자료상 부족

## 언제 선택하나

**선택**
- 시크릿이 코드와 설정에 흩어져 있어 중앙 통제가 필요한 조직
- 클라우드와 DB 자격 증명을 동적으로 발급, 회수하려는 환경
- 애플리케이션 암호화를 표준화하려는 개발 조직
- 시크릿 접근 감사가 규제로 요구되는 산업

**회피**
- 운영 전담 인력이 없어 복잡도를 감당하기 어려운 소규모 팀
- 단순 키 저장만 필요한 경우 (클라우드 기본 KMS로 충분)
- 기기 위험 기반 접근 통제가 1차 과제인 경우

## 도입 고려사항

**라이선스.** 오픈소스 버전과 Enterprise, HCP 관리형을 제공합니다. Enterprise는 국내 파트너를 통한 도입이 일반적입니다.

**데이터 리전.** 자체 호스팅을 선택하면 시크릿 저장 위치를 조직이 직접 통제할 수 있습니다. HCP 관리형의 리전과 데이터 보관 위치는 도입 시 확인이 필요합니다.

**CSAP.** Vault의 한국 클라우드 보안인증(CSAP) 획득 여부는 공개 자료에서 확인되지 않았습니다. 공공 도입 검토 시 최신 인증 현황을 직접 확인하세요.

**망분리.** 자체 호스팅이 가능해 망분리 환경 적용 여지가 큽니다. 구체 구성은 별도 검토가 필요하며, 이번 검토에서 국내 적용 사례는 확인하지 못했습니다.
